# 🚀 دليل نشر منصة خشير (Kshare)

أهلاً بك! 👋 هذا هو الدليل الكامل لإطلاق منصة **خشير** — سوق الشراكات السعودي على طريقة Kickstarter — من جهازك إلى الإنترنت. اتبع الخطوات بالترتيب، وخلال أقل من ساعة ستكون منصتك حيّة وتستقبل أول المهتمين بإذن الله! 💪

## 🧰 التقنيات المستخدمة

| المكوّن | الخدمة |
|---------|--------|
| قاعدة البيانات | Supabase (PostgreSQL) |
| الاستضافة والدوال | Vercel (Serverless — Node.js 20) |
| بوابة الدفع | Moyasar (ميسر — بوابة دفع سعودية) |
| البريد الإلكتروني | Resend |

## 💰 ملخص التسعير السريع

| البند | السعر |
|-------|-------|
| نشر فكرة (رسوم الإعلان) | **99 ريال** |
| فتح تواصل (رسوم المهتم) | **25 ريال** |
| أرباح صاحب الفكرة | **5 ريال** عن كل فتح تواصل |
| نقطة التعادل لصاحب الفكرة | **20 مهتم** (20 × 5 = 100 ريال ≈ رسوم النشر) |

---

## 1️⃣ الخطوة الأولى: تشغيل قاعدة البيانات

1. افتح لوحة تحكم [Supabase](https://supabase.com/dashboard) وادخل على مشروعك.
2. من القائمة الجانبية اختر **SQL Editor**.
3. انسخ محتوى ملف `kshare_schema.sql` (الذي أنشأناه مسبقاً) والصقه في المحرر، ثم اضغط **Run**.
4. تأكد من نجاح التنفيذ: افتح **Table Editor** وتحقق من وجود:
   - جدول `listings` (الأفكار المنشورة)
   - جدول `unlocks` (عمليات فتح التواصل)
   - جدول `transactions` (سجل المدفوعات)
   - العرض `public_listings` (النسخة العامة الآمنة من الإعلانات)

✅ إذا ظهرت الجداول الثلاثة والعرض، فقاعدة بياناتك جاهزة!

---

## 2️⃣ الخطوة الثانية: رفع الكود على GitHub

من داخل مجلد المشروع `kshare_api`، نفّذ الأوامر التالية في الطرفية (Terminal):

```bash
git init
git add .
git commit -m "🚀 خشير v1.0"
```

بعدها:

1. ادخل على [github.com](https://github.com) وأنشئ مستودعاً جديداً (New Repository) باسم **kshare** — بدون README أو أي ملفات إضافية.
2. اربط المستودع وارفع الكود (استبدل `USERNAME` باسم حسابك):

```bash
git remote add origin https://github.com/USERNAME/kshare.git
git push -u origin main
```

✅ حدّث صفحة GitHub وستجد ملفاتك كلها هناك.

> ⚠️ **تنبيه:** تأكد أن الكود لا يحتوي على أي مفاتيح سرية قبل الرفع! المفاتيح مكانها Vercel فقط (الخطوة 4).

---

## 3️⃣ الخطوة الثالثة: النشر على Vercel

1. ادخل على [vercel.com/new](https://vercel.com/new).
2. اختر **Import Git Repository** واربط حساب GitHub إذا لم يكن مربوطاً.
3. اختر مستودع **kshare**.
4. في إعدادات الاستيراد:
   - **Framework Preset:** اختر `Other`
   - **Root Directory:** اتركه فارغاً (الجذر)
5. اضغط **Deploy** وانتظر دقيقة تقريباً. ☕

✅ سيعطيك Vercel رابطاً مثل `https://kshare.vercel.app` — احفظه، ستحتاجه في الخطوات القادمة.

> ملاحظة: النشر الأول قد يعمل بالواجهة فقط، لكن الدوال (API) لن تعمل بشكل صحيح حتى تضيف متغيرات البيئة في الخطوة التالية — هذا طبيعي تماماً.

---

## 4️⃣ الخطوة الرابعة: إعداد متغيرات البيئة في Vercel

بعد اكتمال النشر، ادخل على مشروعك في Vercel ثم:

**Project Settings ← Environment Variables**

أضف **جميع** المتغيرات التالية (لكل المتغيرات اختر البيئات الثلاث: Production و Preview و Development):

| المتغير | من أين تحصل عليه | مثال |
|---------|------------------|------|
| `MOYASAR_SECRET_KEY` | [moyasar.com](https://moyasar.com) ← لوحة التحكم ← **مفاتيح API** ← انسخ **المفتاح السري** (Secret Key) | `sk_test_...` |
| `RESEND_API_KEY` | [resend.com](https://resend.com) ← **API Keys** ← اضغط **Create API Key** وانسخ المفتاح فوراً (لن يظهر مرة أخرى) | `re_...` |
| `SUPABASE_URL` | Supabase ← **Project Settings** ← **API** ← انسخ **Project URL** | `https://xxx.supabase.co` |
| `SUPABASE_SERVICE_KEY` | Supabase ← **Project Settings** ← **API** ← انسخ مفتاح **service_role** (وليس anon!) | `eyJ...` |
| `NEXT_PUBLIC_SITE_URL` | رابط مشروعك على Vercel من الخطوة السابقة | `https://kshare.vercel.app` |

> ## 🔐 **تحذير أمني بالغ الأهمية**
>
> **المفاتيح `MOYASAR_SECRET_KEY` و `RESEND_API_KEY` و `SUPABASE_SERVICE_KEY` يجب ألا توضع أبداً — تحت أي ظرف — داخل ملفات HTML أو JavaScript في الواجهة، ولا تُرفع أبداً إلى GitHub. مكانها الوحيد هو Environment Variables في Vercel.**
>
> **مفتاح `service_role` تحديداً يتجاوز كل صلاحيات الحماية (RLS) في قاعدة البيانات — تسريبه يعني وصولاً كاملاً لبياناتك! وتسريب مفتاح ميسر السري يعني إمكانية إجراء عمليات مالية باسمك.**

## 5️⃣ الخطوة الخامسة: إعادة النشر بعد إضافة المتغيرات

متغيرات البيئة لا تُطبَّق على النشر القديم — لازم إعادة نشر:

1. من مشروعك في Vercel، ادخل على تبويب **Deployments**.
2. بجانب آخر نشر (الأحدث في القائمة)، اضغط قائمة النقاط الثلاث **⋯** ثم اختر **Redeploy**.
3. أكّد بالضغط على **Redeploy** مرة أخرى.

✅ بعد اكتمال إعادة النشر، صارت الدوال (API) تقرأ المفاتيح وتعمل بكامل طاقتها.

---

## 6️⃣ الخطوة السادسة: تحديث رابط الموقع في ميسر (Moyasar)

حتى يقبل ميسر عمليات الدفع القادمة من موقعك:

1. ادخل على [moyasar.com](https://moyasar.com) ← لوحة التحكم ← **الإعدادات**.
2. أضف رابط موقعك على Vercel (مثل `https://kshare.vercel.app`) إلى قائمة **روابط الرجوع المسموحة** (Allowed Callback URLs).
3. احفظ التغييرات.

✅ بدون هذه الخطوة قد تُرفض عمليات الدفع أو يفشل الرجوع بعد الدفع، فلا تتجاوزها!

---

## 7️⃣ الخطوة السابعة: اختبار النظام 🧪

حان وقت التأكد أن كل شيء يعمل! افتح رابط موقعك على Vercel وجرّب التالي:

### أ. تحميل الإعلانات
- تأكد أن الصفحة الرئيسية تعرض الإعلانات من Supabase (عبر `public_listings`) بدون أخطاء.
- افتح Console في المتصفح (F12) وتأكد من عدم وجود أخطاء حمراء.

### ب. اختبار نشر فكرة (99 ريال)
- جرّب المسار كاملاً: تعبئة نموذج نشر فكرة ← الانتقال لصفحة الدفع ← إتمام الدفع.
- استخدم **بطاقة ميسر التجريبية**:

| الحقل | القيمة |
|-------|--------|
| رقم البطاقة | `4111 1111 1111 1111` |
| تاريخ الانتهاء | `12/25` |
| CVV | `123` |

- بعد الدفع، تأكد أن الفكرة ظهرت في الموقع وأن صفاً جديداً أُضيف في جدولي `listings` و `transactions`.

### ج. اختبار فتح تواصل (25 ريال)
- من أي إعلان منشور، جرّب زر **فتح تواصل** وأكمل الدفع بنفس البطاقة التجريبية.
- تأكد من ظهور بيانات التواصل بعد الدفع، ومن إضافة صف في جدول `unlocks`.

### د. البريد الإلكتروني
- تأكد من وصول رسائل التأكيد عبر Resend (لصاحب الفكرة وللمهتم).
- إذا لم تصل، راجع تبويب **Logs** في لوحة Resend وتبويب **Logs** للدوال في Vercel.

✅ إذا نجحت كل الاختبارات — مبروك، النظام يعمل من الألف إلى الياء! 🎉

---

## 8️⃣ الخطوة الثامنة: التحويل لوضع الإنتاج 🟢

آخر خطوة قبل استقبال مدفوعات حقيقية:

### 1. مفاتيح ميسر الحقيقية
- من لوحة ميسر، فعّل الوضع الحقيقي (Live) واحصل على المفتاح السري الحقيقي (يبدأ بـ `sk_live_...`).
- في Vercel ← **Settings** ← **Environment Variables**، حدّث قيمة `MOYASAR_SECRET_KEY` بالمفتاح الحقيقي بدل التجريبي.
- لا تنسَ **Redeploy** بعد التحديث (كما في الخطوة 5).

### 2. توثيق النطاق في Resend
- من لوحة Resend ← **Domains** ← أضف نطاقك `kshare.sa` واتبع تعليمات إضافة سجلات DNS (SPF و DKIM).
- بعد التوثيق، صار بإمكانك الإرسال من عنوان احترافي مثل `no-reply@kshare.sa` بدل نطاق الاختبار.

### 3. النطاق المخصص (اختياري لكن مُوصى به)
- إذا ربطت نطاقاً مخصصاً (مثل `kshare.sa`) بمشروعك في Vercel من **Settings ← Domains**:
  - حدّث قيمة `NEXT_PUBLIC_SITE_URL` إلى الرابط الجديد (مثل `https://kshare.sa`).
  - أضف الرابط الجديد أيضاً في إعدادات ميسر (كما في الخطوة 6).
  - ثم **Redeploy**.
